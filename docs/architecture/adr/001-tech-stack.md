# ADR-001: 前端技术栈选择

**日期**: 2025-11-25
**状态**: 已接受
**作者**: Stephen

---

## 背景

YBP (永麦 BIM Platform) 需要一个网页可视化展示系统，用于展示 BIM 族库管理和工程量计算的业务逻辑。该系统主要用于：

1. 向需求方（永麦）展示业务流程
2. 辅助开发方（易达）理解需求
3. 未来可能作为实际产品的前端基础

---

## 决策

采用以下技术栈：

| 技术 | 版本 | 用途 |
|------|------|------|
| React | 19 | UI 框架 |
| Vite | 7 | 构建工具 |
| Tailwind CSS | 3 | 样式系统 |
| React Router | 7 | 路由管理 |
| React Flow | 12 | 流程图可视化 |
| pnpm | - | 包管理器 |

**不使用**：
- TypeScript
- Tailwind CSS v4

---

## 理由

### React 19
- 最新稳定版本，性能优化
- 社区生态成熟，资源丰富
- 团队熟悉度高

### Vite 7
- 极速开发体验（HMR 即时生效）
- 构建速度快
- 配置简单

### Tailwind CSS v3（而非 v4）
- v3 版本成熟稳定
- v4 当时处于 Beta 阶段，存在不稳定风险
- 项目周期短，不宜承担技术风险

### 不使用 TypeScript
- 项目规模小（3 页面）
- 开发周期短（原型阶段）
- 快速迭代优先

---

## 后果

**正面后果**：
- 开发效率高，快速迭代
- 学习曲线低
- 社区支持好

**负面后果**：
- 无类型检查，大型项目可能有问题
- 后续重构为 TypeScript 需要额外工作

---

## 替代方案

### 方案 1: Vue + Vite
- **优点**: 更轻量，上手快
- **缺点**: 团队 React 经验更丰富
- **未采用原因**: 团队熟悉度

### 方案 2: Next.js
- **优点**: SSR 支持，功能丰富
- **缺点**: 项目不需要 SSR，过于复杂
- **未采用原因**: 过度工程化
