## 🎯 核心 Slash Commands

| 命令 | 用途 | 频率 |
|------|------|------|
| `/start` | 恢复项目记忆 | 每次会话开始 |
| `/checkpoint` | 阶段性保存进度 | 完成功能后 |
| `/end` | 每日结束 | 每天结束时 |
| `/weekly` | 每周文档优化 | 每周日 |
| `/monthly` | 每月归档 | 每月初 |
| `/audit` | 标准健康检查 | 每周 |
| `/deep-audit` | 全面深度审计 | Phase 完成后 |
| `/fix` | ESLint 自动修复 | 按需 |

## 📍 定位说明

### 与标准 4 层架构的关系

```
docs/
├── ai-context/      # 第 1 层：AI 上下文（项目特定）
├── development/     # 第 2 层：开发文档（项目特定）
├── architecture/    # 第 3 层：架构文档（项目特定）
├── project/         # 第 4 层：项目文档（项目特定）
└── guides/          # ⚠️ 参考文档（通用，可跨项目复用）
```

### 如何复用到其他项目

1. **复制整个 guides 目录**到新项目的 `docs/guides/`
2. **复制 `.claude/commands/`** 目录到新项目
3. **根据项目调整**技术栈配置表中的变量
4. **创建项目特定的** `CONTEXT.md` 和 `CURRENT.md`

### 注意事项

- 这些文档是**通用模板**，不包含项目特定信息
- 项目特定的配置应在 `ai-context/CONTEXT.md` 中记录
- 如需修改 Slash Commands，应直接编辑 `.claude/commands/` 下的文件
