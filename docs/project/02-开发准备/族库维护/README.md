## 待解决问题

### 1. 参数同步机制

**问题描述**：

系统参数与 Revit 参数的同步流程：

```
系统新增参数 → API 写入族 → 建模员修改 → 上传回系统
```

**待明确**：

- [ ] 冲突处理：系统和 Revit 参数值不一致时，以哪边为准？
- [ ] 版本控制：参数配置变更后，已下载的族如何处理？
- [ ] 批量更新：是否支持批量更新已放置族实例的参数？

---

### 2. 清单库版本管理

**初步方案（黄增沛 2025-11-25）**：

- 架构：沿用单价库 + 清单的方式
- 版本策略：版本精确到清单级别（去除大版本）
- 核心识别：清单编码 + 清单名称

**待确认**：

- [ ] 对比 YDC、汉堡王、奥乐齐的清单结构
- [ ] 确认版本精确到清单级别的具体实现
- [ ] YBP 业态的扩展识别码定义

---
