# YBP 网页设计总览

> Google 风格的现代暗色主题设计方案

**创建日期**: 2025-11-25
**最后更新**: 2025-12-10
**设计风格**: Google-inspired Material Design 3 Dark Theme

---

## 一、设计背景

### 1.1 项目现状

- 需求讨论完成
- 原型设计完成
- 网页展示系统开发完成
- 易达评审中，准备开发

### 1.2 网页目标

| 目标 | 说明 | 优先级 |
|-----|------|--------|
| **汇报演示** | 向领导展示项目价值和进展 | P0 |
| **会议讨论** | 与易达讨论时的可视化参考 | P0 |
| **知识沉淀** | 结构化保存讨论成果 | P1 |

### 1.3 设计原则

- **精简**: 只展示关键信息，不是文档搬运
- **美观**: 参考 Google 设计语言，高品质视觉
- **动效**: 流畅的动画增强体验

---

## 二、页面架构

### 2.1 三页结构

```
YBP 网页
│
├── 首页 (/)
│   ├── Hero 区域（项目标题）
│   ├── 核心流程（族→品目→清单→工程量）
│   ├── 项目进度
│   └── 快速入口
│
├── 业务逻辑 (/logic)
│   ├── 四种匹配规则
│   └── 三个典型案例
│
└── 问题追踪 (/issues)
    ├── 统计卡片
    └── 问题列表
```

### 2.2 路由配置

| 页面 | 路由 | 组件 | 说明 |
|-----|------|------|------|
| 首页 | `/` | HomePage | 项目概览 |
| 业务逻辑 | `/logic` | LogicPage | 匹配规则 + 案例 |
| 问题追踪 | `/issues` | IssuesPage | 待讨论问题 |

## 三、视觉设计系统

### 3.1 设计风格

- **整体**: Google-inspired 现代暗色主题
- **参考**: Google Chrome、Google One、About Google
- **特点**: 大圆角、渐变背景、玻璃效果、流畅动画

### 3.2 配色方案

```
表面色（Surface）- 带蓝调的深灰，非纯黑
├── surface-dim:    #111827  // 最深背景
├── surface:        #1f2937  // 主背景
└── surface-bright: #374151  // 亮表面

主色调（Primary）
├── primary:        #60a5fa  // 柔和蓝色
├── secondary:      #a78bfa  // 紫色
└── tertiary:       #34d399  // 绿色

文字色（Text）
├── on-surface:         #f9fafb  // 主文字
└── on-surface-variant: #d1d5db  // 次要文字

边框色
├── outline:         #4b5563
└── outline-variant: #374151

状态色
├── error:   #f87171
├── success: #34d399
└── warning: #fbbf24
```

### 3.3 字体规范

```
英文字体：Times New Roman, Georgia
中文字体：PingFang SC, Microsoft YaHei
字体族：  serif
```

### 3.4 圆角规范

```
├── rounded-2xl: 1rem (16px)
├── rounded-3xl: 1.5rem (24px)  // 主要卡片
└── rounded-full: 9999px        // 按钮、标签
```

### 3.5 渐变色

```
流程节点渐变：
├── 族:     from-blue-500 to-blue-600
├── 品目:   from-purple-500 to-purple-600
├── 清单:   from-emerald-500 to-emerald-600
└── 工程量: from-amber-500 to-amber-600

状态渐变：
├── 待讨论: from-amber-500 to-orange-500
├── 讨论中: from-blue-500 to-indigo-500
└── 已解决: from-emerald-500 to-teal-500
```

## 四、动画系统

### 4.1 入场动画

```css
/* 淡入上移 */
.animate-fade-up {
  animation: fadeUp 0.6s ease-out forwards;
}

@keyframes fadeUp {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```

### 4.2 滚动动画

```javascript
// Intersection Observer 实现
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add('visible')
      }
    })
  },
  { threshold: 0.1, rootMargin: '0px 0px -50px 0px' }
)
```

### 4.3 交互动画

```css
/* 悬浮上浮 */
.hover-lift:hover {
  transform: translateY(-4px);
  box-shadow: 0 12px 40px rgba(0, 0, 0, 0.3);
}

/* 展开/收起 */
.transition-all {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

### 4.4 特效

```css
/* 背景渐变网格 */
.gradient-mesh {
```

## 五、组件结构

### 5.1 项目结构

```
src/
├── components/
│   └── Layout.jsx        // 布局（Header + Footer）
│
├── pages/
│   ├── HomePage.jsx      // 首页
│   ├── LogicPage.jsx     // 业务逻辑
│   └── IssuesPage.jsx    // 问题追踪
│
├── App.jsx               // 路由配置
├── main.jsx              // 入口
└── index.css             // 全局样式 + 动画
```

### 5.2 Layout 组件

- 玻璃效果 Header（固定定位）
- 渐变 Logo
- 响应式导航（桌面/移动端）
- 简洁 Footer

### 5.3 数据内联

所有页面数据直接定义在组件文件中，不使用独立数据文件。

## 六、响应式设计

### 6.1 断点设置

```
├── sm:  640px   // 手机横屏
├── md:  768px   // 平板
├── lg:  1024px  // 笔记本
└── xl:  1280px  // 桌面
```

### 6.2 布局适配

| 断点 | 导航 | 卡片列数 | 内容宽度 |
|-----|------|---------|---------|
| < 640px | 汉堡菜单 | 1列 | 全宽 |
| 640-1024px | 顶部导航 | 2列 | 全宽 |
| > 1024px | 顶部导航 | 2-4列 | max-w-6xl |
